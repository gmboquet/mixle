# Mixle 0.8.0 release-preparation checklist

This is the authoritative package-local checklist for 0.8.0. The release contract is defined by
`docs/release-readiness.rst`; supporting decisions, risks, review protocols, and stage gates are in this
directory. The former link to an ignored `notes/` worklist was removed because a release process must be
available from a clean clone.

Preparation branch: `release-prep/0.8.0` → `release/0.8.0`.

**Do not create `v0.8.0`, publish artifacts, merge the preparation PR, or mark final sign-off while any
`HOSTED`, `EXTERNAL`, or `POST-RELEASE` gate below remains.** The tag is deliberately outside this work.

## Status vocabulary

- `COMPLETE` — implemented and checked locally on the preparation branch.
- `HOSTED` — implementation exists; the exact final commit must pass the named hosted job.
- `EXTERNAL` — requires a human reviewer, external user, registry, or designated hardware.
- `EXCLUDED` — claim or operation is intentionally omitted, with the limitation stated.
- `POST-RELEASE` — can run only after publication.

Local validation commands are individually capped at 30 seconds. Broad suites and the full Sphinx build belong
in hosted CI; timing out locally is not converted into a pass.

## Release decision

The code and documentation preparation can be complete while the release is **not yet publishable**. The final
PR may be opened when local preparation evidence is recorded. Publication requires the exact-tip hosted jobs,
external review/sign-off, and post-merge artifact rehearsal. No checklist status authorizes creating the tag
early.
