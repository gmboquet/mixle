# Migrating to 0.8.0

Status: development draft; 0.8.0 is not yet published.

## Input validation

Quantitative semantics, calibration, posterior schemas, MinHash controls, law
discovery, and sequential-design budgets reject malformed, non-finite, or
out-of-range values earlier. Callers that relied on silent acceptance must
validate inputs and handle ValueError at the public boundary.

## Empirical law discovery

Candidate forms are selected on a validation split and the winner is confirmed
on an untouched holdout split. The result adds selection_r2 and n_selection.
ranking contains validation scores; holdout_r2 determines passed. Code that
interpreted ranking values as final confirmation scores must update.

## Sequential-design budgets

max_rounds means the maximum adaptive acquisitions after the initial fit, so a
completed trace can contain max_rounds + 1 fitted states. Negative,
non-integral, and boolean budgets are rejected.

## Imports and tests

The root lazy namespace translates only a genuinely absent requested submodule
to AttributeError; a missing dependency inside an existing module now
propagates with its original cause. Pytest collects both supported filename
conventions, which may expose previously skipped tests in custom automation.

## Rollback

Pin the previous supported release and restore artifacts produced by its
compatible schemas. Do not suppress new validation errors without confirming
that the prior accepted behavior was scientifically meaningful.
